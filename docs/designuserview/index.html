Design User View

Design User View

js/src/designcomps/designuserview.js

Landing Page for Design side, presents list of available templates and option to create a template

State Variables

  • template_rows
    Stores the meta data for each template: {name, email, created, lastModified, id}
    Populated by loadData()

  • filter
    Text from the filter input, used in render(). Only template rows whose name or emailfields contain this text will be shown
    Updated by anonymous function, in the onChange handler of the filterInput field in render()

  • order_by
    One of {“Titile”, “AuthorEmail”, “Created”, or “Modified}
    Must match the field names in firestore
    used when requests are loaded in loadData()
    Set via anonymous function in the onClickhandler for teh order_by dropdown

  • order_by_direction
    One of {“asc”, “desc”}
    used when requests are loaded in loadData()
    Set via anonymous function in the onClick handler for the order_by_direction dropdown

Functions

  • loadData()
  • deleteTemplate()
  • convertTemplateInfoToRow(template, index)
    Template is an item from state.template_info and index is the item’s index in the state.template_info array
    If state.filter is included in the name or author email, a <li> is generated with info about the list. Every column, but the trash (delete) links to the template’s edit page.

Rendering

  • Maps template metadata from state.template_info to <li>'s using convertTemplateInfoToRow
